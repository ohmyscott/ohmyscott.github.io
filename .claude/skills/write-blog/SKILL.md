# 写博客（write a blog）
收到输入（title、category、tags、references、notes）后，按流程读取参考并生成完整 Markdown 博文，严格使用第一人称并在末尾列出参考文章链接

## Instructions
提供清晰、可执行的逐步指导，按下面流程自动生成博客。生成的博客必须满足以下硬性要求：
- 前置元数据（frontmatter）必须统一使用如下 YAML 语法格式（严格一致）。
- 博文全文必须以第一人称视角撰写（使用“我”）。
- 若有参考文章，必须在文末“参考文章”处列出完整链接。
- 原文或笔记中的专业词汇可以用反引号或引号标注，并在同处或文末解释；必要时保留英文词汇并加注中文说明。

## 统一 frontmatter 模板（必须使用）
```yaml
---
date: 2025-11-30T23:15:00+08:00
title: "{{title}}"          # 博客标题（字符串）
date: {{date}}            # YYYY-MM-DD
category: ["{{category}}"]  # 分类（字符串）数组，逗号分隔
tags: [{{tags}}]            # 标签数组，逗号分隔
slug: "{{slug}}"            # 博客文章URI，方便SEO
draft: false                # 是否草稿
description: "{{description}}"      # 一段简短摘要（一句到两句）
ShowToc: true
TocOpen: true
---
```

## 写作流程（逐步）
1. 接收输入：
    - title, category, tags（数组或逗号分隔）、参考文章列表（URLs，可为空）、我的笔记摘要（自然语言文本）。
2. 读取参考文章：
    - 按顺序访问每个 URL，提取文章标题、作者、发布日期、关键论点与直接引用（如有）。
    - 标记遇到的专业术语与定义。
3. 合并笔记摘要与参考要点：
    - 将用户笔记作为优先上下文，补充参考文章中有价值的事实、数据或引用。
    - 对冲突处，注明来源并取最可靠信息。
4. 生成文章结构（默认骨架）：
    - frontmatter（使用上面模板）
    - lead（开头 1-2 段，说明写作目的或背景，与主要结论，第一人称）
    - 正文：分若干小节，每节用小标题，逻辑清晰，必要时使用列表、代码块（如果涉及代码或配置）
    - 专业词汇说明（可选，小节）
    - 结论（可选，第一人称）
    - 参考文章（完整链接）
5. 专业词汇与引用处理：
    - 直接引用原文时使用 blockquote，并在引用后以括号注明来源 URL（或脚注格式）。
    - 专业词汇在首次出现时用反引号标注，例如：`API`（应用程序编程接口，Application Programming Interface），随后可以仅保留英文或中文视上下文决定。
6. 风格与长度控制：
    - 写作风格：专业、易读、简洁；句子偏短，逻辑紧凑。
    - 始终使用第一人称（“我”）表述观点与经验。
    - 尽量把复杂概念拆成要点或示例解释。
7. 输出与校验：
    - 输出为完整 Markdown 文件（含 frontmatter）。
    - 在文末列出“参考文章”标题与原始链接；若引用了参考文中的具体段落，应在引用处加括号标注来源。
    - 检查语法、标点与一致性（如标签格式、日期格式）。

## 辅助信息
技术主题类型的文章，你在构思文章结构时可以借鉴：SCQA原则（结构化表达工具，芭芭拉·明托在《金字塔原理》提到）：
- S:situation (情景，背景) --问题发生的背景
- C:confilt(冲突) --产生的冲突
- Q:question （问题）--面临的问题
- A: answer (回答) --自己的回答，解决方案。

## 注意事项（简洁）
- 保持中英文术语一致：首次出现给出中文解释与英文原文（或反之）。
